# プライバシーポリシー — CLASS Calendar

最終更新日: 2026年5月28日

## 概要

CLASS Calendar は、東京理科大学の CLASS ポータルから時間割を取得し、Google カレンダーに登録する Chrome 拡張機能です。

## 取り扱うデータについて

本拡張機能は、開発者が運営する外部サーバーへ個人データを送信・収集・保存することは一切ありません。

### 端末内で扱うデータ

| データ | 保存場所 | 目的 |
|---|---|---|
| 学期の開始日・終了日 | Chrome ローカルストレージ（端末内のみ） | 次回起動時の入力省略 |
| イベントの色設定 | Chrome ローカルストレージ（端末内のみ） | 次回起動時の選択状態の復元 |
| CLASS時間割カレンダーの ID | Chrome ローカルストレージ（端末内のみ） | カレンダーの重複作成防止 |
| 時間割データ（授業名・教室・時限） | メモリのみ（ポップアップを閉じると消去） | ポップアップへの表示とカレンダー登録 |
| Google OAuth アクセストークン | メモリのみ（ポップアップを閉じると消去） | Google カレンダー API の呼び出し |

## 通信先

本拡張機能が通信する外部サービスは以下の2つのみです。

- **CLASS ポータル**（`class.admin.tus.ac.jp`）— 時間割の取得
- **Google カレンダー API**（`googleapis.com`）— カレンダーへのイベント登録・削除、「CLASS時間割」カレンダーの作成

その他の第三者サービスへのデータ送信は行いません。

## Google アカウントへのアクセスについて

Google ログイン時に要求する権限は `calendar`（カレンダーの作成・イベントの作成/読み取り/削除）です。Google の認証情報は本拡張機能に保存されません。

この権限は以下の目的にのみ使用されます。

- **カレンダー作成**：「CLASS時間割」カレンダーが存在しない場合の初回作成
- **登録**：「CLASS時間割」カレンダーへのイベント作成
- **重複チェック**：登録前に同じ授業の予定が既に存在するか確認するための読み取り
- **削除**：本拡張機能で登録した予定の削除（履修変更時などに使用）

読み取りは本拡張機能が登録した予定（内部タグで識別）のみを対象とし、それ以外の予定の内容を取得・参照することはありません。
